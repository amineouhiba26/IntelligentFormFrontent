Cahier des Charges pour Claude – Formulaire Dynamique “Le Nexus Connecté”
1. Objectif

Développer un Formulaire d’Interaction Dynamique pour le site de la Nuit de l’Info 2025 qui :

Guide l’utilisateur vers une mission spécifique.

Adapte automatiquement les champs du formulaire selon la mission choisie.

Assure la sécurité, l’accessibilité et la compatibilité multi-terminaux.

Fournit un retour personnalisé intégrant le nom de l’utilisateur, sa mission, l’année en cours et le thème de l’événement.

2. Fonctionnalités attendues
2.1 Sélection de mission

Menu déroulant ou boutons radio pour choisir la mission :

“Établir le Contact”

“Offrir un Don”

“Rejoindre la Guilde des Bénévoles”

“Demander des Informations”

Chaque sélection déclenche dynamiquement l’affichage des champs correspondants.

2.2 Adaptation des champs

Établir le Contact : nom, email, message.

Offrir un Don : nom, email, montant (préconfiguré et libre), récurrence (ponctuelle, mensuelle).

Rejoindre les Bénévoles : nom, email, compétences, disponibilités.

Demander des Informations : nom, email, sujet de la demande.

2.3 Validation et sécurité

Validation côté client et serveur :

Champs obligatoires

Format email

Montant positif pour les dons

Protection anti-spam : CAPTCHA ou Honeypot.

Transmission sécurisée via HTTPS.

2.4 Personnalisation et retour utilisateur

Après soumission, afficher une page de confirmation dynamique :

Mentionner le nom de l’utilisateur (si fourni).

Rappeler la mission choisie.

Insérer l’année en cours et le thème de l’événement.

Exemples :

“Salutations, [Nom] ! Ton message a bien été acheminé. Nos agents de support te répondront sous peu.”

“Merci [Nom] ! Ton don de [Montant] € contribue directement à [impact du don].”

2.5 Responsive design

Le formulaire doit être adapté à tous les terminaux : ordinateurs, tablettes, smartphones.

3. Critères techniques

Technologies front-end : HTML, CSS/SCSS, JavaScript ou framework Angular/React (au choix).

Back-end : Node.js / Spring Boot / Python Flask (selon stack existante).

API endpoints : pour gérer la soumission des données selon la mission.

Sécurité : HTTPS, validation côté serveur, protection anti-spam.

Accessibilité : WCAG 2.1 (labels, tab navigation, contrastes).

Personnalisation dynamique : le LLM doit générer les champs et les messages de confirmation selon les inputs de l’utilisateur.

4. Livraison attendue

Formulaire fonctionnel intégré au site web.

Page de confirmation dynamique personnalisée.

Documentation expliquant :

Les champs générés pour chaque mission

Les règles de validation

L’API utilisée pour la soumission des données

Comment accéder et tester le formulaire sur le site.

5. Critères d’évaluation

Innovation / originalité du formulaire : 20 pts

Normes d’accessibilité respectées : 20 pts

Intégration du thème Nuit de l’Info 2025 dans le retour utilisateur : 20 pts

UX intuitive et efficace : 10 pts

Sécurité et robustesse du formulaire : 10 pts
